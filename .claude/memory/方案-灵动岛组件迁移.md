# 灵动岛组件迁移到 reef-widgets 框架方案

## Context

**问题**: 当前存在两套独立的灵动岛渲染体系：
- `framework/reef-widgets/` — 基于 `Widget` trait 的组件框架，有骨架但视觉保真度低
- `dynamic_island/` + `framework/reef-ui/` — 生产应用实际使用的管线（场景→表现→视觉计划→渲染命令），功能完整但未框架化

**目标**: 把 `reef-ui` 中的完整视觉能力补入 `reef-widgets`，消除四层间接转换，让 `dynamic_island` 切到统一的 Widget 渲染范式。

## 总体策略

分 5 个阶段，每阶段独立可验证，不影响现有功能：

---

### 阶段 1: 补全 Card Widget 视觉规格

将 `reef-ui/card_visual_spec/` (1300+ 行) 的完整视觉能力补入 `reef-widgets/src/card.rs`。

**具体改动**:
- `card.rs`: 增加 `CardVisualStyle::Pending` 变体 (reef-ui 有 8 种，当前只有 7 种)
- `card.rs`: 替换硬编码颜色为 reef-ui 的完整颜色映射（每样式: 外壳填充、边框、徽章状态色、徽章源色、重音色、前缀色、文本色）
- `card.rs`: 增加 `CardVisualBodyRole` 枚举 (Assistant/User/Tool/Plain/ActionHint)，每角色独立前缀色+文本色
- 新增 `src/card_body.rs`: `CardBodySpec` / `CardBodyLineSpec`，处理 body line 的绘制
- 新增 `src/card_badge.rs`: 状态徽章和源徽章，区分圆形 vs. 彩色药丸样式，6 种源背景色 (Claude/Codex/Gemini/Feishu 等)
- `card.rs`: 工具药丸增加 description 布局、每工具色调 (bash=绿/edit=蓝/read=黄/grep=紫/agent=橙)
- `card.rs`: 操作提示药丸使用 reef-ui 的 metric 常量
- `card.rs`: 设置行增加值徽章 Widget (边框+填充圆角矩形+居中文本)

**关键文件**: `framework/reef-ui/src/native_panel_ui/card_visual_spec/mod.rs` (参考颜色/规格常量)

---

### 阶段 2: 补全 CompactBar + Chrome 过渡

**具体改动**:
- `compact_bar.rs`: 集成 `CompactShoulder` (左/右肩部，带进度动画)
- `compact_bar.rs`: 活动计数跑马灯 (数字滚动动画，活>0 时绿色)
- `compact_bar.rs`: 操作按钮 (设置/退出)，带缩放+位置过渡动画
- `compact_bar.rs`: 头部文本侵扰处理 (操作按钮可见时缩小可用头部宽度)
- `compact_bar.rs`: 完成发光叠加层支持
- `island_widget.rs`: Chrome 过渡逻辑 (分隔符可见性、肩部进度、操作按钮缩放/位置、折叠 alpha 退出)

**关键文件**: `framework/reef-ui/src/native_panel_ui/visual_plan/compact.rs`, `action_button_visual_spec.rs`

---

### 阶段 3: 补全吉祥物完整功能

**具体改动**:
- `mascot.rs`: 增加 `MascotPose::MessageBubble` / `MascotPose::WakeAngry` 姿态
- 新增 `src/mascot_bubble.rs`: 信息气泡 Widget (圆角矩形+3 个椭圆点)
- 新增 `src/mascot_badge.rs`: 完成徽章 Widget (轮廓+填充圆角矩形+标签文本)
- 新增 `src/mascot_sprite.rs`: 精灵动画系统 (manifest 解析、关键帧选择、帧定时)
- `mascot.rs`: 增加阴影支持 (shadow_opacity/shadow_radius，跟随身体运动)
- `mascot.rs`: 增加运动系统 (offset_x/offset_y/shell_alpha/debug_mode 可视化)

**关键文件**: `framework/reef-ui/src/native_panel_ui/mascot_visual_spec.rs`, `mascot_sprite_spec.rs`

---

### 阶段 4: 补全动画和交互层

**具体改动**:
- 新增 `src/animation.rs`: Widget 属性插值器 (宽度/高度/肩部进度/drop 进度/卡片进度)，基于 `Instant` 时间线
- 新增 `src/interaction.rs`: 指针区域定义、命中测试、事件路由
- `island_widget.rs`: 卡片堆栈交错揭示动画 (进入/退出 stagger、缩放+高度插值、translate_y+淡色内容揭示)
- `island_widget.rs`: 堆栈溢出滚动/裁剪 (当卡片总高度 > 容器时)
- `island_widget.rs`: 卡片内容裁剪区域 (ClipStart/ClipEnd 包装)

**关键文件**: `framework/reef-ui/src/native_panel_ui/animation_scheduler.rs`, `visual_plan/expanded.rs`

---

### 阶段 5: 桥接 dynamic_island → reef-widgets

**具体改动**:
- 新增 `dynamic_island/src/island_widget_bridge.rs`: `fn build_island_widget(state: &PanelState, snapshot: &RuntimeSnapshot) -> IslandWidget`
- 修改 `dynamic_island/src/native_panel_renderer/mod.rs`: 精简 facade，删除 scene→presentation→visual→command 重导出
- 修改 Windows 绘制器: 添加消耗 `VisualPrimitive` (来自 WidgetHost::render()) 的路径 (当前消耗 `NativePanelRenderCommandBundle`)
- 动画帧写入: `NativePanelAnimationFrameScheduler` 每帧更新 `IslandWidget` 字段 → `WidgetHost::mark_dirty()` + `render()`
- 交互层: 保留现有轮询/悬停/点击子系统在外围，将视觉输出路由切换到 WidgetHost
- 清理: 删除不再需要的模块引用 (runtime_render_payload.rs 等可合并或删除)

**关键文件**: `dynamic_island/src/native_panel_renderer/mod.rs`, `windows_native_panel/paint_backend.rs`, `host_runtime.rs`

---

## 不变部分

以下模块在工作完成后保持不变：
- Win32 窗口创建和 Shell 生命周期
- 模式生命周期和胶囊窗口管理 (mode_lifecycle.rs)
- 状态机 (state_machine.rs)
- 预览宿主结构 (preview_host.rs)
- 应用设置、显示枚举
- 更新服务、通知音

## 验证方式

**每阶段验证**:
- `cargo build -p reef-widgets` 编译通过
- `cargo test -p reef-widgets` 测试通过
- 阶段 1-4: 更新 `examples/hello-reef` 验证新特性渲染正确
- 阶段 5: `cargo build -p dynamic_island` 编译通过，运行 `island-ui.rs` 预览二进制验证 UI 与迁移前一致

**最终验收**:
- 灵动岛所有 UI 状态 (Compact/Expanded/Hidden、各卡片样式、吉祥物姿态、设置面板) 渲染与迁移前视觉一致
- 交互行为不变 (悬停展开/折叠、点击派发、设置切换)
- 性能无退化 (帧率不低于迁移前)
